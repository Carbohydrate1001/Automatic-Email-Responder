Model Configuration

Parameter Value
Model GPT-4o-mini (OpenAI)
Response Format JSON Object
Temperature 0.0 (Business Gate) / 0.1 (Category)
Max Retries 3 attempts with exponential backoff
Circuit Breaker 5 failures → 60s recovery timeout
Business Gate Threshold ≥ 0.85 confidence
Fallback Strategy Rule-based classification on API failure

Confidence Interpretation

Range Level Action
0.0 - 0.5 Low Confidence Requires manual review
0.5 - 0.75 Medium Confidence Manual review recommended
0.75 - 0.90 High Confidence Consider auto-processing
0.90 - 1.0 Very High Confidence Suitable for auto-processing

Classification Process

1 📧 Email Received
System receives new email, extracts subject and body
Input: subject, body, sender, received_at
Source: Microsoft Graph API (Inbox)
2 🔍 Rule-Based Pre-Filter
Fast rule check: contains non-business keywords (no API call)
Non-business hints: promotion, limited offer, newsletter, storage full
Business hints: order, logistics, refund, invoice, tracking
❓ Has non-business keywords AND no business keywords?
✅ YES
Category: non_business
Confidence: 0.98
Skip API calls
❌ NO
Continue to
Business Gate
3 🚪 Business Gate Check (LLM)
Use GPT-4o-mini to determine if email is a real business service request
Prompt: BUSINESS_GATE_PROMPT
Output: {is_business_related, confidence, reasoning}
Retry: Max 3 retries with circuit breaker
❓ is_business_related = false AND confidence ≥ 0.85?
✅ YES
Category: non_business
Use gate confidence
End classification
❌ NO
Continue to
Category Classification
4 🎯 Category Classification (LLM)
Use GPT-4o-mini to classify email into specific business category
Prompt: CATEGORY_PROMPT
Output: {category, confidence, reasoning}
Temperature: 0.1 (deterministic)
5 📊 Rubric-Based Scoring (Optional)
Use structured scoring criteria to assess classification quality, provide explainable confidence
Dimensions: keyword_match (25%), intent_clarity (30%), context_completeness (25%), exclusion_confidence (20%)
Formula: confidence = weighted_score / 3.0
Fallback: Use original LLM confidence if rubric fails
6 💾 Save to Database
Save classification result to database, including category, confidence, reasoning, and scoring details
Table: emails
Fields: category, confidence, reasoning, classification_rubric_scores (JSON)
Status: Initially set to 'pending_review'
Key Features: ① Three-layer filtering mechanism (rule → business gate → category classification) reduces API calls | ② Retry mechanism and circuit breaker ensure system stability | ③ Rubric scoring provides explainable confidence | ④ Fallback strategy ensures system works even when API fails
